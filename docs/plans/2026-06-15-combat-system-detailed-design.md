# 战斗系统详细设计

## 一、骰子数据架构

### 配置层（静态）
```
骰面模板库
├── ID: attack_light   → { 效果: "造成5点伤害", 开销: "行动力2" }
├── ID: attack_heavy   → { 效果: "造成10点伤害", 开销: "行动力3" }
├── ID: attack_crit    → { 效果: "造成20点伤害", 开销: "行动力4" }
└── ...

骰体预设配置
├── ID: attack_dice_D4 → { 骰面: [attack_light, attack_heavy, attack_miss, attack_crit] }
└── ...
```

### 运行时数据（动态）
```
玩家的攻击骰实例
├── 骰体配置: attack_dice_D4  (引用配置ID)
├── 骰面集合: [attack_light, attack_heavy, attack_miss]  (初始3面)
├── 自定义骰面: [attack_bleed_01]  (成长奖励新增的4th面)
└── 骰面强化数据: (详见存档设计文档 → 待补充)
```

> **数据持久化说明**：
> 骰面强化、自定义骰面等玩家成长数据将在【存档系统设计文档】中详细定义。
> 链接：`docs/plans/YYYY-MM-DD-save-system-design.md`（待创建）

## 四、行动力系统

> 精力为十维属性之一；属性不产出骰子，只缩放公式与作门槛。详见 [属性系统设计](2026-06-25-attribute-system-design.md)。

### 基础公式
```
计算行动力 = 精力属性（阶梯分段） + 行动力回复骰结果
最终行动力 = min(计算行动力, 单位最大行动力上限)
```

### 精力属性阶梯（配置驱动）
- 每个单位有独立的阶梯配置模板
- 示例：精力1-10→5行动力，11-20→8行动力，21-30→12行动力
- 可通过特殊机制修改

### 修正因素
- **轻功养成** → 提升精力属性阶梯
- **装备重量** → 降低最大行动力上限（详见【装备系统设计文档】）
- **行动力回复骰** → 天赋/才华系统提供，骰子组+骰面模板驱动

### 装备重量接口
```
最大行动力 = 单位模板最大值 - 装备超重惩罚
超重惩罚规则：详见【装备系统设计文档】
链接：docs/plans/YYYY-MM-DD-equipment-system-design.md（待创建）
```

### 修正示例
```
角色A（轻功10级，无超重装备）：
  精力15 → 阶梯行动力8
  骰面回复+3 → 计算11
  最大行动力10 → 最终10

角色A（轻功10级，装备超重）：
  精力15 → 阶梯行动力8
  骰面回复+3 → 计算11
  最大行动力修正为9 → 最终9
```

## 五、移动与战棋

> **设计语境**：战棋网格以「走位＝战术核心」为基础，保持中武偏写实、凡人尺度，不依赖超自然能力——与 [武侠世界观总纲](2026-06-26-worldview-design.md) 第一、七节一致。

### 动态战场生成

#### 核心逻辑
```
1. 世界地图 = M×N 格点矩阵，每个格有可通行性配置
2. 触发战斗时：
   → 获取所有参战单位的世界坐标
   → 圈出最小矩形边界
   → 检查是否满足战斗配置的最小尺寸
   → 不足则四向扩展（遇边界停止）
3. 最终战场 = 可能不对称的"不规则矩形"
```

#### 最小战场尺寸配置
```
优先级：战斗事件配置 > 敌方单位性质 fallback > 全局默认

战斗事件配置：
  - 可配置固定战场位置（剧情战斗、Boss战等）
  - 可配置最小战场尺寸
敌方单位性质 fallback：
  - 小规模遭遇（1-2敌人）：8×8
  - 中型战斗（3-5敌人）：10×10
  - 大型战斗（6-10敌人）：12×12
  - Boss战：15×15（或更大）
全局默认：8×8
```

#### 特殊Case：剧情战斗预设战场
```
某些剧情战斗需要指定特殊位置的战场：

场景示例：
  - 竹林决斗：指定竹林特定区域，战场15×10
  - Boss战：Boss房间固定位置，战场20×15
  - 城门守卫战：城门口区域，战场12×18

实现方式：
  - 战斗事件配置中标记"use_fixed_battlefield: true"
  - 配置战场起止坐标（如 from: [50, 30], to: [70, 45]）
  - 系统直接使用该区域，不进行动态生成
```

#### 示例场景
```
世界地图 100×100，玩家在(95, 45)遭遇敌人(97, 48)
圈出矩形：(95,45) 到 (97,48) = 3×4
战斗配置最小尺寸：8×8
扩展尝试：
  - 向右扩展：2格 → 到达边界(100)，实际只能扩展到100
  - 向上/下/左：正常扩展
最终战场：(89, 41) 到 (100, 49) = 11×9（右侧受限）
```

#### 边界战术特性
边界不补偿规则能创造独特战术环境：
- 悬崖边战斗：一侧无法移动，压迫感强
- 墙角战斗：空间狭小，近战单位优势
- 河边战斗：无法过河，远程单位优势

### 移动回溯机制

#### 简单回溯规则
```
移动阶段：
  - 消耗移动骰进行移动
  - 到达目标位置

行动阶段开始前：
  - 玩家可选择"回溯" → 回到回合开始位置
  - 或"执行一个非移动行为" → 确认位置，进入行动阶段

限制：只能回溯到回合开始位置，不能部分回溯
```

#### 交互流程
```
1. 投骰阶段 → 获得移动骰（如：移动骰面5）
2. 移动阶段 → 移动1~5格到位置A
3. 预览位置：在新位置展开操作预览
4. 玩家选择：
   - 新的操作 → 锁定位置A，执行下一个行动阶段
   - 取消 → 回到原点，可重新移动（仍受移动骰点数限制）
```

#### 策略意义
- 降低走位失误的惩罚
- 鼓励玩家大胆尝试走位
- 快节奏，无需复杂计算

### 移动距离计算

#### 骰面直接配置
```
每个移动骰面直接配置移动距离：

骰面ID: move_3
  效果: { type: MOVE, distance: 3 }
  开销: { actionPoints: 0 }

骰面ID: move_5
  效果: { type: MOVE, distance: 5 }
  开销: { actionPoints: 0 }

骰面ID: sprint_8
  效果: { type: MOVE, distance: 8 }
  开销: { actionPoints: 1 }
```

#### 移动距离是上限
```
骰面配置移动距离 = 最大可移动格数
玩家可自由选择移动 0~最大距离 之间的任意格数

示例：
  - 移动骰点数5 → 可走0、1、2、3、4或5格
  - 玩家根据战术需求决定实际移动距离
  - 未使用的移动距离不积累到下回合
```

#### 配置驱动优势
- 策划可精确设计不同移动骰的强度
- 特殊骰面可配置额外效果（如sprint消耗行动力）
- 易于平衡和调整

### 格子属性（MVP阶段）

#### 最小化属性
```
每个格子只有单一属性：可通行性

可通行：单位可以移动到该格子
不可通行：墙壁、水域等障碍物，单位无法进入

MVP阶段不考虑：
  - 地形类型（草地、水域等）
  - 高度差
  - 地形效果
  - 覆盖物（隐蔽、阻挡）
```

#### 扩展计划（后续迭代）
```
未来可扩展的格子属性：
  - 地形类型：影响移动、防御
  - 高度：影响射程、移动
  - 覆盖物：隐蔽、阻挡视线
  - 陷阱/机关
```

## 六、行动执行流程

### 投骰阶段

#### 全部投掷规则
```
1. 系统投出所有骰子（基础骰 + 联动骰）
2. UI展示所有骰面结果
3. 玩家查看结果，规划本回合行动
```

#### 投骰示例
```
角色A的骰池：
  - 攻击骰D6 → 投出4（重击）
  - 移动骰D6 → 投出5（快速移动）
  - 休息骰D4 → 投出2（小恢复）
  - 剑法联动骰D4 → 投出剑法技能

UI展示：
  [攻击骰:4] [移动骰:5] [休息骰:2] [剑法骰:突刺]
```

#### 骰子开始效果
```
投骰完成后，执行所有骰子的"开始效果"：
  - 某些骰子有回合开始时触发的能力
  - 示例：防御骰的"本回合防御+2"开始效果
```

---

### 行动阶段

#### 行动选择流程
```
1. 玩家从菜单选择行动类型（移动/攻击/技能/物品/等待）
2. 系统显示该类型可用的骰子
3. 玩家选择具体骰子（移动/攻击/技能需要骰子）
4. 选择目标，确认执行
```

#### 行动类型菜单
```
主行动菜单：
  ├─ 移动
  │   └─ 使用移动骰进行移动
  ├─ 攻击
  │   ├─ 普通攻击（使用攻击骰）
  │   └─ 武功攻击（使用武功骰 + 联动骰）
  ├─ 技能
  │   └─ 特殊技能（消耗行动力，无需骰子）
  ├─ 物品
  │   └─ 使用消耗品
  ├─ 等待（延后行动）
  │   └─ 牺牲当前 50% 行动条，保留当轮骰子结果，暂不行动
  ├─ 查看状态
  └─ 回合结束
```

#### 移动行动
```
移动是回合中可随时执行的行动：

1. 选择"移动"
2. 显示移动骰的可达范围
3. 选择移动方式并移动

多输入方式支持：
  - 鼠标点击/拖拽
  - 键盘方向键
  - 手柄摇杆/方向键

移动可以执行多次：
  - 每次移动消耗一个移动骰
  - 可以在攻击前后移动
  - 移动骰用完后无法再移动
```

#### 鼠标移动操作
```
点击模式：
  1. 显示移动骰的可达范围（如：5格内所有格子高亮）
  2. 玩家点击目标格子
  3. 系统自动计算最短路径并移动

拖拽模式：
  1. 玩家拖拽角色图标
  2. 实时显示路径预览
  3. 释放到目标格子，完成移动
```

#### 键盘移动操作
```
1. 玩家用方向键/WASD移动角色
2. 每次移动1格
3. UI显示剩余可移动距离
4. 按Enter/Space确认移动位置
```

#### 手柄移动操作
```
1. 左摇杆/方向键：移动光标选择格子
2. 右摇杆：快速移动光标
3. A键（或X）：确认移动
4. B键（或圆）：取消
5. 触觉反馈：边界碰撞时轻微震动
```

#### 骰子选择与组合
```
选择"攻击"后：
  - UI显示可用的攻击骰
  - 点击某个攻击骰
  - 如果有可联动的武器骰/武功骰，提示"是否联动？"
  - 确认后选择目标

示例：
  1. 选择"攻击"
  2. 显示：[攻击骰:4] [剑法骰:突刺]
  3. 点击剑法骰
  4. 提示："是否联动剑武器骰？[是] [否]"
  5. 选择目标敌人，确认执行
```

#### 目标选择
```
单目标技能：
  - 点击目标单位
  - 显示伤害预览
  - 确认执行

范围技能：
  - 高亮显示范围内的所有格子
  - 点击格子确定释放位置
  - 显示受影响的所有单位
  - 确认执行
```

#### 联动骰组合规则
```
联动骰必须与基础骰配合使用：

武器骰 + 攻击骰：
  - 改变攻击类型（切割→穿刺）
  - 可能增加额外效果

武功骰 + 攻击骰：
  - 扩展攻击范围
  - 添加特殊效果
  - 额外消耗行动力

组合顺序：
  - 先选基础骰
  - 系统提示可联动的骰子
  - 选择联动骰（可选）
  - 确认组合
```

#### 等待（延后行动）

核心机制：
```
等待是行动阶段内可主动选择的战术行为：

  - 代价：牺牲当前 50% 行动条（行动力）
  - 效果：保留当轮投出的所有骰子结果
  - 状态：暂不行动，延后本回合的行动决策
```

**策略意义**
- 以 50% 行动条为代价换取决策弹性，保留骰子结果等待更优使用时机
- 配合联动骰、战场局势与队友节奏，灵活调整行动顺序
- 在不确定或不利局面下暂缓出招，规避风险

**使用限制**
- 与"结束回合"不同：等待只暂缓行动，并触发持有骰的蓄力效果，不触发留用效果和回合结算

### 结束阶段

#### 手动确认机制
```
1. 玩家点击"结束回合"按钮
2. 系统提示："有未使用的骰子，确认结束？"
3. 确认后触发留用效果
4. 回合结束
```

#### 自动结束配置
```
玩家可在设置中配置"自动结束回合"：

选项：
  - 手动确认（默认）：每次需手动确认结束回合
  - 延迟自动：行动完成后，N秒后自动结束
  - 立即自动：行动完成后立即结束回合

延迟自动的交互：
  - 行动完成后，5秒倒计时
  - 期间显示"取消"按钮
  - 点击取消 → 返回行动阶段
  - 倒计时结束 → 回合结束
```

#### 回合结算流程
```
确认结束回合后：
  1. 触发所有未使用骰子的"留用效果"
     - 示例：休息骰 → 恢复气血
  2. 所有骰子回收到骰池
  3. 行动力清零
  4. 更新Buff持续时间
  5. 回合结束，轮到下一个单位
```

#### Buff持续机制
```
回合结束时更新所有Buff：
  - 持续回合数 -1
  - 到期的Buff移除
  - 新生效的Buff触发
```

## 行动执行流程总结

### 完整回合流程
```
0. 进入战斗（仅一次）
   ├─ 获得所有骰子的入场效果

1. 投骰阶段
   ├─ 系统投出所有骰子
   ├─ UI展示骰面结果
   └─ 执行骰子开始效果

2. 行动阶段（循环执行，直到行动力耗尽或主动结束）
   ├─ 选择行动类型：
   │   ├─ 移动（消耗移动骰）（未执行下一个非移动行为时可以Undo）
   │   ├─ 攻击（消耗攻击骰 + [可选联动骰] + 行动力）
   │   ├─ 武学（消耗行动力 + 基础骰 + 武学联动骰 + [可选联动骰]）
   │   ├─ 物品（消耗物品）
   │   ├─ 等待（牺牲 50% 行动条，保留骰子触发蓄力效果，暂不行动）
   │   └─ 查看状态
   ├─ 选择具体骰子（如适用）
   ├─ 选择联动骰（如适用）
   ├─ 选择目标
   └─ 执行效果

3. 结束阶段
   ├─ 手动确认或自动结束
   ├─ 触发留用效果
   ├─ 回收骰子
   └─ 更新Buff状态
```

#### 流程特点
```
- 移动不是独立阶段，是可随时执行的行动
- 玩家可自由安排行动顺序：
  - 先移动再攻击
  - 先攻击再移动
  - 移动→攻击→再移动
  - 多次移动（如果有多个移动骰）
- 行动力限制总行动次数
- 骰子数量限制特定行动次数
```
